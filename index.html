Process Scheduling Information

First-Come, First-Served (FCFS)

The simplest scheduling algorithm where processes are executed in the order they arrive in the queue. It's fair but can lead to the "convoy effect" where short processes wait behind long ones.

Priority Scheduling

Processes are executed based on priority levels. Higher priority processes are executed first. Can lead to starvation of lower priority processes if not implemented with aging.

Round Robin (RR)

Each process gets a small unit of CPU time (time quantum), and if not completed, it returns to the queue. This ensures fair CPU distribution and good response time.

Shortest Job First (SJF)

Processes with the shortest execution time are executed first. Optimal for minimizing average waiting time but requires knowing process execution time in advance.

Key Performance Metrics

  • Turnaround Time: Total time from submission to completion
  • Waiting Time: Time spent waiting in the ready queue
  • Response Time: Time from submission until the first response
  • Throughput: Number of processes completed per unit time
  • CPU Utilization: Percentage of time CPU is busy
